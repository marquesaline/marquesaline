APIs estão em todo lugar hoje em dia. Qualquer aplicação que se comunica com outra, seja um app mobile consumindo dados do backend, ou dois serviços conversando entre si, provavelmente tem uma API no meio. 

Então entender como elas funcionam é essencial. Nesse post a gente vai ver o que é uma API, o que é REST e como isso tudo aparece no Rails.

## O que é uma API?

API é a sigla pra Application Programming Interface, que basicamente é uma solução que permite que diferentes serviços e aplicações se comuniquem entre si.

Esses serviços não precisam conhecer como o outro funciona internamente, só precisam saber como enviar e receber informações um do outro. 

É por isso que é chamada de interface, porque a API funciona como uma ponte que vai permitir a interação entre sistemas diferentes.

Existem vários tipos de API, mas a que a gente vai conversar é sobre a mais comum, a API REST.

### API REST

REST é uma sigla pra Representational State Transfer e não é uma tecnologia em si, é um estilo arquitetural. Isso quer dizer que é um conjunto de princípios que guiam como uma API deve ser projetada pra ser simples, escalável e fácil de usar.

Quando a gente diz que uma API é RESTful, está dizendo que ela segue esses princípios. Alguns deles têm a ver com como o HTTP é usado, outros com como os dados são organizados e trafegados.

#### Recursos

No REST, tudo gira em torno de recursos. Um recurso é qualquer coisa que você quer expor pela sua API, pode ser um usuário, um produto, um pedido, o que fizer sentido pro seu sistema.

Cada recurso tem sua própria URL, e é por ela que você acessa e manipula esse recurso (dado). Por exemplo, `/users` representa a coleção de usuários e `/users/1` representa um usuário específico que possuí o `id` 1. Essa URL é o endereço do recurso, e a ação que você quer fazer nele vai depender do verbo HTTP que você usar.

#### Verbos HTTP

O REST usa os verbos do HTTP pra definir o que você quer fazer com um recurso. A ideia é que cada verbo tem um significado bem definido: `GET` é pra buscar, `POST` é pra criar, `PUT` ou `PATCH` é pra atualizar e `DELETE` é pra remover.

Isso torna a API bem previsível, porque só de olhar pra URL e pro verbo da requisição já dá pra entender o que ela faz.

Ahh e fique atento pra manter esse padrão tá?! Os verbos realmente são muito importante e é essencial que a gente use de acordo com a ação que vai ser executada. 

#### Stateless

Stateless significa que cada requisição feita pra API precisa ter tudo que o servidor precisa pra respondê-la. O servidor não guarda nenhum estado entre uma chamada e outra.

Na prática, isso quer dizer que se você precisa de autenticação, por exemplo, o token precisa ir em toda requisição. O servidor não lembra que você já se autenticou antes, cada requisição começa do zero pra ele.
